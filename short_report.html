1 Project Purpose & Description

A Network Intrusion Detection System (NIDS) monitors network traffic and identifies malicious activity in real time. With cyberattacks growing in frequency and sophistication, automated detection using machine learning has become essential in modern cybersecurity.

This project trains three classifiers on the NSL-KDD benchmark dataset to classify each network connection as either Normal or an Attack. It produces visual reports and provides an interactive command-line interface for live predictions.

Why is this useful? Network administrators can integrate such a system to automatically flag suspicious connections, reducing the burden of manual log inspection and accelerating incident response.

3 Project Workflow

1
Load Dataset NSL-KDD TXT files are downloaded automatically (125,973 training / 22,544 test records, 41 features each)
2
Preprocess Multi-class labels → binary (Normal=0 / Attack=1). Categorical columns label-encoded. All features scaled with StandardScaler.
3
Train Models Three classifiers trained on preprocessed data: Logistic Regression, Decision Tree (max_depth=10), Random Forest (100 trees).
4
Evaluate & Compare Accuracy, precision, recall, F1-score computed on KDDTest+. Best model selected automatically.
5
Save & Visualize Best model saved to models/. Six charts saved to outputs/ as PNG files.
6
Interactive Prediction CLI User enters network feature values → model predicts Normal or Attack with confidence score.

4 Code Explanation

main.py — Entry Point

Parses command-line arguments (--train, --predict, --all) and calls the appropriate pipeline functions. Contains the ASCII banner.

train.py — Training Pipeline

Handles everything from data download to model saving. Key functions: download_dataset(), load_dataset(), preprocess(), train_models(), evaluate_models(), save_best_model(), and six individual plot_*() functions.

predict.py — Prediction CLI

Loads the saved model from disk. Offers a menu: (1) run predefined sample inputs, (2) manually enter feature values, or (3) exit. Returns prediction label + confidence percentage.

# Example: run training
py main.py --train

# Example: run interactive prediction
py main.py --predict

# Example: train then predict in one command
py main.py --all

6 Why README.md and requirements.txt Matter

README.md

The README is the front page of any open source project. It answers: What does this project do? How do I install it? How do I use it? Without a clear README, even brilliant code is unusable by others. On platforms like GitHub, the README renders automatically — it is often the first (and only) thing a visitor reads.

requirements.txt

This file lists every Python library the project depends on, with version constraints. Running pip install -r requirements.txt recreates the exact environment on any machine in seconds. This guarantees reproducibility — one of the core principles of open source software — and eliminates the infamous "works on my machine" problem.
